此版本开始完成用户注册功能

通过此版本,我们要了解用户如何将信息提交到服务端
以及服务端如何通过请求解析用户在页面输入的信息并进行处理
这个操作在request解析的时候进行,因此这是一个通用操作
这个操作做好以后,无论用户请求什么样的业务,传递数据request
都可以处理好并交给处理业务的环节使用

注册流程:
1:用户请求注册界面
2:在注册页面用户输入注册信息
3:点击注册按钮提交注册
4:服务器处理请求
5:响应注册结果页面(成功或者失败)

要点:
1:理解页面中form表单的使用
2:Get请求提交数据的URL格式
3:服务器针对不同请求的不同处理方式
  3.1:请求静态资源(已经实现了)
  3.2:请求一个业务
  
实现
1:在webapps/myweb目录下新建一个注册页面:reg.html
2:提交页面后,发现:URL的抽象路径含有参数
  因此,我们在HttpRequest解析请求行时,需要对抽象路径(URI值进行进一步分析)
  2.1:在HttpRequest中添加3个新属性
  String requestURI: 存请求部分(URI?左侧的内容)
  String queryString:存参数部分(URI?右侧的内容)
  Map parameters:存每一个参数
  
3:在HttpRequest中添加一个方法:parseURI
用于进一步解析URI的值,并在解析请求行的方法
parseRequestLine得到URL后调用parseURI进一步解析
